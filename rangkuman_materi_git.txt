============ Uraian Materi =============

1. intro materi Git
	==> - Git adalah salah satu version control system yang diciptakan oleh Linus Torvalds
	    - GIT mencatat setiap perubahan pada sebuah file di code atau sourcecode
	manfaat git sendiri :
	- Bisa menyimpan seluruh versi sourcecode
	- Berkolaborasi membangun sistem jadi mudah
	- Bisa berkontribusi di proyek open source
	- Memudahkan dalam tracking perubahan
	- Memahami cara deploy modern
	Git ini dibutuhkan ketika :
	- Saat ingin membangun sistem dengan berkolaborasi
	- Membuat versi dari sistem
	- Backup Sourcecode
	- Berkolaborasi dengan programmer dunia
	
2. Fundamental GIT
	Pada materi ini berisikan perintah perintah dasar yang digunakan ketika kita bekerja dengan menggunak GIT :
	- Git Init ==> untuk menginisialisasi repository
	- Git config ==>untuk konfigurasi email dan username git local dengan server yang biasanya digunakan berupa github/gitlab
	- Git Status ==> Untuk mengetahui status file,secara umum status file ini terbagi menjadi 3 :
				1. Untracked
				2. Staging area
				3. Modified
	- Git add ==> untuk memasukkan file ke Stagging area
	- Git Log ==> Untuk riwayat commit
	- Git Commit ==> Untuk mencatat perubahan yang dilakukan
	- Git Clone ==> Untuk mengcopy repo dari server
	- Git Remote ==> Semacam API untuk berinteraksi dari client dengan server
	- Git Push ==> Untuk mengirimkan commit ke server
	- Git Pull ==> Untuk mengambil commit dari server jika ada perubahan di server yang dibuat oleh branc lain
	
3. Branch
	==>Branching adalah membuat cabang dari repositori utama dan melanjutkan melakukan pekerjaan pada cabang yang baru tersebut 
	   tanpa perlu khawatir mengacaukan yang utama.Branch ini dibutuhkan pada saat mengembangkan sebuah fitur baru.terdapat perintah -perintah dasar yang digunakan pada materi branch ini yaitu:
	- Git branch ==> untuk melihat semua branch (local)
	- Git branch --all ==> Untuk melihat semua branch (public)
	- Git branch nama_branch ==> Untuk membuat sebuah branch baru
	- Git checkout nama_branch ==> Untuk berpindah branch
	- Git branch -d nama_branch ==> Untuk menghapus sebuah branch
	- Git merge nama_branch ==> Untuk menggabungkan commit dari 2 branch,relatif terhadap branch yang aktif saat ini, dan untuk melakukan merge harus dipastikan tidak ada file yang belum di commit
